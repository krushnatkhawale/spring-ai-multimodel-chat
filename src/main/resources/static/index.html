⚙️ System Configuration

The fallback engine is initialized with the following order. When a model fails, the gateway immediately issues a retry to the next listed client.

Google Gemini gemini-2.5-flash
OpenAI GPT gpt-4o-mini
Local Ollama llama3

⚙️ Dynamic Fallback Architecture

The fallback mechanism traverses the defined sequence. Exceptions caught from one model client (e.g. due to bad keys, rate limits, or network timeouts) are logged and stored. If a model succeeds, the trace log is appended to the response data payload.

How Fallback Works

At startup, the application creates the required ChatModel beans conditionally based on your classpath starters:

  • OpenAI: Auto-configures an OpenAiChatModel using property prefix spring.ai.openai.
  • Gemini: Auto-configures a GoogleGenAiChatModel using property prefix spring.ai.google.genai.
  • Ollama (Local): Auto-configures an OllamaChatModel using property prefix spring.ai.ollama.

Our custom MultiChatService maps key strings (e.g., "openai", "gemini", "local") to these beans. If a bean is null (not enabled) or throws a runtime exception, the fallback loop intercepts the exception, records it, and immediately forwards the query to the next provider.

🔌 API Endpoints Documentation

Expose local REST operations for system interaction:

POST /api/chat

Submit a prompt to the gateway. The prompt will go through the configured fallback chain.

Request Body:

{
  "prompt": "Explain Quantum Entanglement in 1 sentence",
  "overrideSequence": ["local", "openai"]  // Optional override list
}

Response Payload (Success):

{
  "provider": "openai",
  "response": "Quantum entanglement is a phenomenon where particles become interconnected...",
  "errorsLog": [
    "Failed using provider 'local'. Error: Connection refused"
  ],
  "durationMs": 1420
}

Response Payload (Failure - All Failed):

{
  "error": "All fallback models failed",
  "message": "All configured model providers failed. Errors: Failed using provider 'gemini'. Error: 401 Unauthorized | Failed using provider 'openai'. Error: 401 Unauthorized..."
}
GET /api/config

Retrieve the default fallback sequence configured in the application.yml file.

Response:

[
  "gemini",
  "openai",
  "local"
]

🚀 Quick Start Guide

Follow these steps to run the application and set up custom parameters:

1. Environment Configuration:

Provide keys as shell exports before running the service:

export GEMINI_API_KEY="AIzaSyYourGeminiStudioKey"
export OPENAI_API_KEY="sk-proj-YourOpenAiKey"
export OLLAMA_BASE_URL="http://localhost:11434"

2. Launch application:

./gradlew bootRun

3. Edit Fallback List:

Open src/main/resources/application.yml and adjust multichat.fallback-sequence to change priority order.
